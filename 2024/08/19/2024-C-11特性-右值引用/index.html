右值引用

发表于2024-08-19|更新于2024-08-22|C++ 后台开发C++ 11
|阅读量:

右值引用

  • 左值: 存储在内存中,有明确存储空间(可以取地址)的数据
  • 右值: 可以提供数据值的数据(不可以去地址)
  • 引用: 表示别名
  • 注意引用本质就是 const 指针,本身就不可以修改指向,加上 const 表示不可以修改它的值
  • 各种概念的介绍如下:

右值引用的作用

  • 右值引用的作用就是延长右值存活时间
  • 比如函数的返回值是一个临时变量,如果不是用右值引用,那么就会调用拷贝构造函数浪费资源
  • 所以可以利用右值引用接受或者提供移动构造函数的方式来是的接受的值可以获取即将释放的变量的资源(将亡值)
  • 两个概念:
    • 纯右值: 非引用返回的临时变量,运算表达式产生的临时变量,原始字面量和 lambda 表达式
    • 将亡值: 与右值引用相关的表达式,比如 T&& 类型函数的返回值,std::move函数的返回值

&& 的特性

  • && 并非总是表示右值引用,有时候可以作为不确定引用类型的推导,比如使用模板或者 auto (T&&,auto&&)
  • 推导规则:
    • 通过右值推导 && 或者 auto&& 得到的就是一个右值引用
    • 通过非右值(右值引用,左值,左值引用,常量右值引用,常量左值引用)推导T&& 或者 auto&& 得到的就是一个左值引用类型
  • const T&& 就是一个右值引用,不需要推导

右值传递

  • 注意对于右值引用,如果有名字,那么右值引用就会被当成左值引用处理
  • 左值和右值都是独立于他们的类型之外的,右值引用类型可能是右值或者左值

转移和完美转发

move 函数

  • 使用 std::move方法可以把左值转换为右值,使用这一个函数并不可以移动任何东西,而是和移动构造函数一样都具有移动语义,将对象的状态或者所有权从一个对象转移到另外一个对象,只能是转移,没有内存拷贝(就相当于状态的转变)
  • move 的作用:
    • 可以利用 move 初始化右值引用
    • 当一个对象不再使用的时候,可以使用 move 函数进行资源的转移,防止资源浪费
